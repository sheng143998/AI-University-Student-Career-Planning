# 职引 AI - Auth 模块接口文档

## 模块概述

用户认证相关接口，包括登录、注册、退出登录，以及用户信息编辑（修改昵称、密码、上传头像）。

---

## 接口列表

| 接口编号 | 接口名 | 方法 | 路径 | 说明 | 鉴权 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 1.1 | 用户登录 | POST | `/api/user/login` | 登录获取 Token | 不需要 |
| 1.2 | 用户注册 | POST | `/api/user/register` | 用户注册 | 不需要 |
| 1.3 | 退出登录 | POST | `/api/auth/logout` | 使 token 失效 | 需要 |
| 1.4 | 获取用户信息 | GET | `/api/user/info` | 获取用户当前信息（用于编辑回显） | 需要 |
| 1.5 | 编辑用户信息 | PUT | `/api/user/edit` | 统一编辑接口（昵称、密码、头像） | 需要 |

---

## 1.1 用户登录

### 1.1.1 基本信息

请求路径：`/api/user/login`

请求方式：`POST`

接口描述：该接口用于用户登录，验证账号密码后返回 JWT Token 及用户基本信息。

---

### 1.1.2 请求参数

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| username | string | 必须 | 用户名（邮箱） |
| password | string | 必须 | 登录密码（明文） |

请求参数样例：
```json
{
  "username": "user@example.com",
  "password": "your_password"
}
```

---

### 1.1.3 响应数据

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 返回的数据 |
| &#124;- token | string | 非必须 | JWT Token |
| &#124;- expires_in | number | 非必须 | Token 过期时间（秒） |
| &#124;- user | object | 非必须 | 用户基本信息 |
| &#124;&#124;- id | number | 非必须 | 用户 ID |
| &#124;&#124;- name | string | 非必须 | 用户名 |

响应数据样例：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "token": "<jwt_token>",
    "expires_in": 86400,
    "user": {
      "id": 1001,
      "name": "张三"
    }
  }
}
```

---

### 1.1.4 错误响应

| HTTP 状态码 | 场景 | JSON 响应样例 |
| :--- | :--- | :--- |
| `400` | 请求参数格式错误 | `{"code": 0, "msg": "参数格式错误"}` |
| `401` | 账号或密码错误 | `{"code": 0, "msg": "账号或密码错误"}` |

---

## 1.2 用户注册

### 1.2.1 基本信息

请求路径：`/api/user/register`

请求方式：`POST`

接口描述：该接口用于新用户注册，创建用户账号。

---

### 1.2.2 请求参数

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| username | string | 必须 | 用户名（邮箱） |
| password | string | 必须 | 登录密码 |
| name | string | 非必须 | 昵称/姓名 |

请求参数样例：
```json
{
  "username": "user@example.com",
  "password": "your_password",
  "name": "张三"
}
```

---

### 1.2.3 响应数据

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 返回的数据 |
| &#124;- registered | boolean | 非必须 | 是否注册成功 |

响应数据样例：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "registered": true
  }
}
```

---

### 1.2.4 错误响应

| HTTP 状态码 | 场景 | JSON 响应样例 |
| :--- | :--- | :--- |
| `400` | 请求参数格式错误 | `{"code": 0, "msg": "参数格式错误"}` |
| `409` | 用户名已存在 | `{"code": 0, "msg": "用户名已存在"}` |

## 1.3 退出登录

### 1.3.1 基本信息

请求路径：`/api/auth/logout`

请求方式：`POST`

接口描述：该接口用于用户退出登录，使当前 Token 失效（可选功能）。

---

### 1.3.2 请求参数

参数说明：

本接口无需请求参数，通过 Header 中的 Authorization 传递 Token。

---

### 1.3.3 响应数据

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 返回的数据 |
| &#124;- ok | boolean | 非必须 | 是否退出成功 |

响应数据样例：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "ok": true
  }
}
```

---

### 1.3.4 错误响应

| HTTP 状态码 | 场景 | JSON 响应样例 |
| :--- | :--- | :--- |
| `401` | 未登录或 token 失效 | `{"code": 0, "msg": "未登录"}` |

---

## 1.4 获取用户信息

### 1.4.1 基本信息

请求路径：`/api/user/info`

请求方式：`GET`

接口描述：该接口用于获取当前登录用户的详细信息，包括昵称、头像等。前端调用此接口后回显到编辑表单，用户在此基础上进行修改。

---

### 1.4.2 请求参数

参数说明：

本接口无需请求参数，通过 Header 中的 Authorization 传递 Token。

---

### 1.4.3 响应数据

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 用户信息数据 |
| &#124;- id | number | 非必须 | 用户 ID |
| &#124;- username | string | 非必须 | 用户名（邮箱） |
| &#124;- name | string | 非必须 | 昵称/姓名 |
| &#124;- avatar | string | 非必须 | 头像 URL |

响应数据样例：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "id": 1001,
    "username": "user@example.com",
    "name": "张三",
    "avatar": "https://example.com/avatars/1001/avatar.jpg"
  }
}
```

---

### 1.4.4 错误响应

| HTTP 状态码 | 场景 | JSON 响应样例 |
| :--- | :--- | :--- |
| `401` | 未登录或 token 失效 | `{"code": 0, "msg": "未登录"}` |

---

## 1.5 编辑用户信息

### 1.5.1 基本信息

请求路径：`/api/user/edit`

请求方式：`PUT`

接口描述：该接口用于统一修改当前登录用户的信息，支持修改昵称、密码、头像。所有字段均为可选，仅传递需要修改的字段。

---

### 1.5.2 请求参数

参数格式：`application/json` 或 `multipart/form-data`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| name | string | 非必须 | 新的昵称/姓名 |
| oldPassword | string | 非必须 | 旧密码（修改密码时必填） |
| newPassword | string | 非必须 | 新密码（修改密码时必填） |
| avatar | file | 非必须 | 头像图片文件（仅 multipart/form-data 支持） |

请求参数样例（JSON - 修改昵称）：
```json
{
  "name": "李四"
}
```

请求参数样例（JSON - 修改密码）：
```json
{
  "oldPassword": "old_password",
  "newPassword": "new_password"
}
```

请求参数样例（Multipart - 上传头像）：
```
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary

------WebKitFormBoundary
Content-Disposition: form-data; name="avatar"; filename="avatar.jpg"
Content-Type: image/jpeg

(binary data)
------WebKitFormBoundary--
```

---

### 1.5.3 响应数据

参数格式：`application/json`

参数说明：

| 参数名 | 类型 | 是否必须 | 备注 |
| :--- | :--- | :--- | :--- |
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 返回的数据 |
| &#124;- updated | boolean | 非必须 | 是否修改成功 |
| &#124;- user | object | 非必须 | 更新后的用户信息 |

响应数据样例：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "updated": true,
    "user": {
      "id": 1001,
      "username": "user@example.com",
      "name": "李四",
      "avatar": "https://example.com/avatars/1001/avatar.jpg"
    }
  }
}
```

---

### 1.5.4 错误响应

| HTTP 状态码 | 场景 | JSON 响应样例 |
| :--- | :--- | :--- |
| `401` | 未登录或 token 失效 | `{"code": 0, "msg": "未登录"}` |
| `400` | 请求参数格式错误 | `{"code": 0, "msg": "参数格式错误"}` |
| `403` | 旧密码错误 | `{"code": 0, "msg": "旧密码错误"}` |

## 前端查询 - 编辑流程说明

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户点击编辑按钮                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  GET /api/user/info                                             │
│  获取用户当前信息（昵称、头像等）                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  前端回显到编辑表单                                              │
│  - 昵称输入框：显示当前 name                                     │
│  - 头像预览：显示当前 avatar                                     │
│  - 密码修改：空（需用户主动填写）                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  用户修改后点击保存                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  PUT /api/user/edit                                             │
│  提交修改数据（仅传递需要修改的字段）                             │
│                                                                 │
│  - 仅修改昵称：{ "name": "新昵称" }                              │
│  - 仅修改密码：{ "oldPassword": "xxx", "newPassword": "yyy" }   │
│  - 仅上传头像：multipart/form-data with avatar file             │
│  - 组合修改：支持上述任意组合                                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  后端返回更新后的用户信息                                        │
│  前端更新本地状态                                                │
└─────────────────────────────────────────────────────────────────┘
```
